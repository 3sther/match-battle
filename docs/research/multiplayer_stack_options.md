# Стек для 2D match-3 battle с realtime PvP - варианты на 2026

Контекст: hobby-игра для 5-7 друзей на Android, один разработчик + AI-ассистент, предыдущая версия (Phaser 3.90 + Vite) потеряна - пишем заново. Игроки в Кыргызстане и России. Критерий номер один - минимальное усложнение и поддержка, не месяцы работы.

## Коротко - рекомендация

**Связка A: Godot 4.6 (нативный APK, клиент и headless-сервер на одном GDScript-коде) + встроенный высокоуровневый multiplayer (WebSocketMultiplayerPeer) + отдельный лёгкий слой аккаунтов/инвентаря (Supabase Auth+Postgres или свой SQLite) + relay-сервер на VPS в России (Timeweb/Selectel).**

Почему именно так - ниже по пунктам, но коротко: нативный Android-экспорт убирает целый класс проблем (WebView, fullscreen, аудио-политики, WebGL-производительность на среднем Android), которых предыдущая web-версия могла и не увидеть, потому что не дошла до продакшена. Годотовский multiplayer из коробки закрывает realtime-синхронизацию хода без стороннего фреймворка. А хостинг в РФ (не Cloudflare, не Hetzner, не Fly.io) снимает единственный по-настоящему серьёзный риск, который нашёлся в исследовании - блокировки/троттлинг западной инфраструктуры для игроков из России.

Альтернатива, если не хочется осваивать новый движок: остаться на Phaser (4 или заново на проверенном 3.90) как PWA по ссылке (без упаковки в APK вообще), Colyseus как self-hosted бэкенд на той же RU-VPS, Supabase только для Auth+Postgres профилей.

### Упаковка веб-клиента в APK (актуально только для Phaser-варианта)

| Способ | Размер / производительность | Подводные камни |
|---|---|---|
| **TWA (Trusted Web Activity)** | Компактный (~800 КБ), быстрый запуск (~1.2с на среднем Android), использует системный Chrome WebView. | Google Play policy 4.3 отклоняет «тонкие обёртки сайта» - TWA без нормального service worker и manifest считается именно такой обёрткой. Для приватного sideload-распространения среди 5-7 друзей это не проблема (Play Store не нужен), но если когда-то захочется публиковаться - нужен полноценный PWA-манифест и офлайн-кэш заранее. |
| **Capacitor** | Тяжелее (~4 МБ, свой WebView-рантайм), запуск чуть медленнее (~1.8с). | Более широкая совместимость, проще добавить нативные плагины (пуши, диплинки) позже. Для чисто игрового WebGL-канваса разница в производительности минимальна. |
| **PWA по прямой ссылке (без APK)** | Ноль усилий на упаковку - просто манифест `display: standalone` + service worker для «Добавить на экран». | Нет иконки в Play Store и списке приложений с автообновлением через маркет, но для 5-7 друзей это не требуется. На Android нужно вручную проверить fullscreen/immersive-режим, разблокировку звука по первому тапу (autoplay policy) и wake lock, чтобы экран не гас во время боя - типичные грабли web-игр на мобильном. |

### WebGL на среднем Android (если остаёмся на Phaser)

WebGL2 в 2026 году покрывает около 98% активных устройств - беспокоиться о совместимости почти не нужно. При аккуратной оптимизации (меньше draw call, простые шейдеры, разумные текстуры) реально получить близко к 60 FPS даже на среднем железе. Единственный системный минус - расход батареи: WebGL съедает заметно больше энергии, чем нативный рендер (WebGPU обещает кратное улучшение, но по состоянию на 2026 год это ещё не мейнстрим-дефолт). Для низкого тикрейта match-3 (не экшен с частицами) это не критично.

**Итог по оси 1:** если минимизировать риск и объём новой работы важнее, чем «остаться в вебе» - Godot нативный APK честно проще, потому что убирает целую категорию проблем разом. Если хочется сохранить максимум переносимого опыта с прошлой версии - Phaser (3.90 ради стабильности, либо 4 ради актуальности) как PWA по ссылке, без упаковки вообще.

## 2. Мультиплеер-бэкенд

Игра пошаговая/полуреалтайм (обмен ходами в match-3-баттле), не шутер - тикрейт низкий, задержки в десятки-сотни мс некритичны. Это сильно расширяет список рабочих вариантов.

Node.js, работает на любом хосте с Node. | Не включены, но `@colyseus/auth` даёт готовые callback-хуки под свою БД (Postgres/Mongo/Redis) - остаётся написать саму схему аккаунтов. | $0 (только хостинг VPS). | Да - лучший баланс «мало кода, много готового» для нашего жанра. |
| **Colyseus Cloud** | То же самое, но managed-хостинг. | Так же, БД своя. | От $15/мес. | Опционально позже, если надоест администрировать VPS - но для 5-7 друзей избыточно. |

Персистентность аккаунтов всё равно выносится в отдельный маленький REST-слой (Supabase или свой Express+SQLite) - подключается по HTTP при старте/окончании сессии, не завязан на тикрейт боя. |

**Итог по оси 2:** для Phaser-варианта - Colyseus self-hosted, он написан именно под наш жанр и не заставляет городить чужую инфраструктуру ради Auth. Для Godot-варианта - его собственный multiplayer API плюс отдельный тонкий слой аккаунтов (Supabase или самодельный). Nakama и managed-облака (Colyseus Cloud, Heroic Cloud) - хорошие продукты, но явно не тот масштаб, под который стоит платить.

Для аудитории «друзья в Кыргызстане и России» это первый фильтр, который стоит применить раньше цены.

## Связки (сборки движок + бэкенд + хостинг)

### Связка A - рекомендация: Godot нативный

- **Клиент:** Godot 4.6, экспорт в APK, sideload на 5-7 устройств (Play Store не нужен - значит и TWA/Capacitor-головная боль отпадает целиком).
- **Realtime:** тот же GDScript-проект, headless-экспорт как authoritative-сервер, `ENetMultiplayerPeer` или `WebSocketMultiplayerPeer` (высокоуровневый multiplayer API, RPC + `MultiplayerSynchronizer`).
- **Аккаунты/инвентарь/гача:** Supabase (Auth + Postgres, бесплатный тариф) как REST-вызов при старте/завершении сессии - не завязан на тикрейт боя, поэтому даже при спорадической нестабильности доступа к Supabase из РФ (см. риски) сама игра не пострадает, только синхронизация профиля отложится. Как fallback - маленький самописный Express+SQLite на той же VPS, если Supabase не устроит по стабильности при тестировании.
- **Хостинг relay-сервера:** Timeweb или Selectel VPS (~10-15 EUR/мес).
- **Сложность:** средняя - новый движок, но полностью убирает web-packaging проблемы; один сервер для realtime и (опционально) БД.
- **Стоимость:** ~10-15 EUR/мес постоянно (VPS) + $0 (Supabase free tier, пока укладываемся в лимиты - для 5-7 игроков это далеко от потолка).

### Связка B - минимальный риск по стеку (сохранить веб-опыт)

- **Клиент:** Phaser (3.90 ради стабильности проверенных паттернов, либо 4 если хочется актуальности и не пугает молодость релиза) + Vite, PWA по прямой ссылке (без упаковки в APK вовсе).
- **Realtime:** Colyseus, self-hosted на Node.
- **Аккаунты/инвентарь:** своя Postgres/SQLite на той же VPS + `@colyseus/auth` хуки, либо тот же Supabase Auth+Postgres, если не хочется писать аутентификацию руками.
- **Хостинг:** Timeweb/Selectel VPS, статика опционально на GitHub Pages если разделять клиент и сервер физически.
- **Сложность:** низкая-средняя - весь стек знаком по прошлой версии, Colyseus добавляет одну новую зависимость, заточенную именно под комнатные игры.
- **Стоимость:** ~10-15 EUR/мес (VPS) + $0 (Supabase, если используется только для Auth/БД).

### Связка C - вообще без своего сервера (быстрее всего запустить, но с оговоркой)

- **Клиент:** Phaser, PWA по ссылке.
- **Realtime + Аккаунты + БД:** всё на Supabase (Realtime Broadcast + Auth + Postgres) - один бесплатный сервис закрывает всю бэкенд-ось разом, вообще без VPS для realtime-части.
- **Хостинг статики:** GitHub Pages.
- **Сложность:** самая низкая - нет сервера, который нужно администрировать вообще.
- **Стоимость:** $0, пока не выйдем за пределы free tier.
- **Оговорка:** модель Broadcast - client-relayed, не authoritative-сервер (валидация хода на клиенте, для игры среди своих друзей это некритично), и есть противоречивые отчёты о нестабильном доступе к Supabase из России без VPN (то работает нормально, то запросы зависают без VPN) - **перед тем как на это закладываться, стоит buквально попросить кого-то из RU-друзей открыть тестовый Supabase-эндпоинт без VPN и посмотреть, действительно ли это проблема именно сейчас**, прежде чем строить на этом всю архитектуру.

## Что стоит проверить в первую очередь (дешёвые spike-и перед тем как закапываться)

1. **Тест доступности из РФ без VPN** для двух кандидатов - Supabase-эндпоинт и выбранная RU-VPS (последняя гарантированно ОК, но убедиться на живом устройстве не помешает). Пять минут, снимает главный неизвестный риск.
2. **Godot: собрать headless-сервер + APK-клиент, погонять ENet/WebSocket multiplayer между двумя телефонами через ту же VPS** - если релей и синхронизация ходов работают за час-два, дальше это уже вопрос игровой логики, а не инфраструктуры.
3. **Проверить пинг Timeweb/Selectel из Бишкека** (не только из Москвы) - для низкого тикрейта match-3 запас большой, но лучше знать заранее, а не гадать.
4. Если всё же тянет к Phaser 4 - собрать один экран (доска match-3, без сети) и прогнать на паре реальных средних Android-телефонов друзей, чтобы увидеть живую производительность WebGL, а не гадать по бенчмаркам из статей.
